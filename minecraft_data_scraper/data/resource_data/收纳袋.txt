简介
收纳袋（Bundle）是一种用于存储其他物品以便管理的物品，通过提示框来作为交互界面。
获取
在 Java 版 中，1.17 版本只能通过指令获取。在加载洞穴与山崖预览数据包后，收纳袋则能够合成。1.18-exp1 版本后，收纳袋可以正常获得和合成了。在 1.21.2 快照24w33a 版本中再次修改了收纳袋的合成配方，使其在前期生存中更容易被获取。
在基岩版中，1.21.30 预览版1.21.30.23 版本以前，只限于开发者版本中使用，在1.21.30.23 版本中，收纳袋为实验性玩法“同捆包”（基岩版的错译），并且可以合成。
用途
收纳袋可以在物品栏中收集储存杂物，最多可以储存等同于 1 组的物品，相当于内部有 64 个槽位，每个单组堆叠上限为 64 的物品占用 1 个槽位，每个上限为 16 的物品占用 4 个槽位，不可堆叠物品、装有蜜蜂的蜂巢或蜂箱（1.21.2 快照24w33a 版本后，仅 Java 版）占用 64 个槽位。当然，收纳袋不能储存潜影盒。
虽然收纳袋本身不可堆叠，但每个收纳袋本身只会占用 4 个槽位，再加上内部已储存的物品。即每个收纳袋最多可以放入 16 个空的收纳袋。
从收纳袋中放入物品，可以将物品拖到收纳袋上方后，使用右键装进收纳袋；或者将收纳袋放在物品上方使用右键。要取出物品，需要右击收纳袋或按住右键在物品栏中拖动。在快捷栏中选中收纳袋时，按下使用键会扔出其中所有的物品，而在物品栏中拖动时，后放入的物品会先释放出来。
在 Java 版 1.21.2 快照24w34a 版本和基岩版 1.21.30 预览版1.21.30.24 版本后，收纳袋的提示框会显示其内容物，最多能查看8个槽位。被高亮的物品（若有）表示右键点击收纳袋时会被取出的物品。如果还未装满，则会在末尾显示数个空格子。如果收纳袋中收纳了超过 8 种不同的物品，那么在第七个物品后面会显示”+…“来表示目前收纳袋里有多少种不同的物品，并且除非拿出前面的物品，否则不能再通过普通方式拿出后方”+…“的物品。可以滑动鼠标滚轮切换下一个取出的物品。
当收纳袋以掉落物的形式被炸毁或烧毁时，其中储存的所有物品都会被释放。
版本信息
收纳袋在 Java 版 1.17 快照20w45a 版本和基岩版 1.18.10 预览版1.18.10.22 版本中加入 ，在基岩版中仅限于开发者版本使用。
在 Java 版 1.17 快照21w19a 版本中，在创造模式物品栏中和收纳袋的合成方式移除，在1.18-exp1 版本中重新加入，在 1.18 快照21w37a 版本中再次在创造模式物品栏中和收纳袋的合成方式移除，在 1.19.3 快照22w42a 版本中重新加入，属于实验性内容，在 Java 版 1.21.2 快照24w33a 版本中更改了合成方式，在24w39a 版本中不再属于实验性内容。
在基岩版 1.21.30 预览版1.21.30.23 版本中，收纳袋属于实验性玩法，在 1.21.40 预览版1.21.40.23 版本中不再属于实验性玩法。
在 Java 版 1.17 快照20w46a 版本、1.21.2 快照24w33a 版本和基岩版 1.21.30 预览版1.21.30.23 版本、1.21.40 预览版1.21.40.23 版本中修改了纹理。
在 Java 版 1.21.2 快照24w38a 版本和基岩版 1.21.40 预览版1.21.40.22 版本中，在实验性内容和实验性玩法中增加了 16 种颜色的变种，都在下一个版本中不再属于实验性内容和实验性玩法。
