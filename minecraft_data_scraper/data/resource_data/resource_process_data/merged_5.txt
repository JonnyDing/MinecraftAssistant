简介
发光地衣（Glow Lichen）是一种有亮度的非固体方块。
自然生成
发光地衣会生成在所有高度的洞穴或洞穴中的含水层中，但不会在深暗之域生成。
获取
发光地衣被挖掘后不会掉落任何物品，但使用剪刀挖掘会掉落本体。
用途
发光地衣的厚度为 0.0625（1/16）格，亮度为 7，可以含水，可以放置在方块完整的表面。在 Java 版 1.17 快照21w03a 版本中发光地衣的厚度为 0.00625（1/160）格。
对发光地衣使用骨粉会使其蔓延到其他可生长的方块表面，即使不能生成也会消耗骨粉。
放进堆肥桶时 50% 的概率使堆肥层数加 1。
在 Java 版 1.17 快照21w03a 版本中，将发光地衣放置在熔岩中会使其变为含水的发光地衣，下界亦是如此，因此可以通过其实现在下界放水。该特性在下一个版本21w05a 版本中被修复。
版本信息

合成器（Crafter）是一种在接收到红石信号时自动合成并丢出产物的方块。
在 Java 版中，合成器需要用任意镐挖掘，但在基岩版中，挖掘最低所需要的工具为石镐。
合成器可以通过合成获得，无法自然生成。
合成器属于红石装置，可以合成物品和作为容器，放置时，输出口是正对着玩家的。对着合成器右击可以打开其 GUI 界面。因为是红石装置，所以会被红石信号激活。GUI 界面
空手单击合成格可以禁用该合成格，再次点击或在该格手动放入物品则启用。
漏斗、投掷器和其他合成器可以与合成器的任何一面互动，将物品转入合成器的未禁用合成格，转入的物品从左到右，从上到下依次填满未禁用合成格。
合成器不能像工作台一样直接被取走，而是会像投掷器一样，向前方掷出产物，如果前方有容器，则会存储在容器中。
如果一个配方会在合成格中留下一些物品，那么这些物品会跟随产物一起被投掷。
在 Java 版中，合成器无法被活塞和黏性活塞推动。

命令方块是一种可以执行命令的方块，使得一些功能可以通过红石信号触发或是自动化完成。
命令方块在Java版1.4.2快照12w32a中加入。
命令方块不会出现在创造模式物品栏中，只能通过/give等命令获取。
命令方块和基岩一样不可被生存模式的玩家破坏，拥有3600000的爆炸抗性，不能被原版任何爆炸所摧毁（仅限无指令）。无论是原版还是MOD。即使是一个全钚核弹或是500*TNT都不行
只有处于创造模式且开启作弊（单人），或是处于创造模式且管理员权限大于等于2才能放置、编辑和破坏命令方块。在服务器中，还需要配置文件中的enable-command-block=true才可以编辑命令方块，否则无论任何权限都无法编辑命令方块。
对放下的命令方块按下使用键（鼠标右键）即可出现命令方块的GUI界面。
在控制台命令一栏可输入命令，命令开头的斜杠（/）是不必须的。加或不加斜杠不会影响命令本身的运行。命令的长度被限制在32500个字符以内。然鹅这么长的限制实在很难突破，想写出一个超过上限的命令都很难
若在命令中使用了@s选择器（1.8+），除非使用/execute命令修改运行环境，否则将不会选中任何目标。
命令方块的本质是模拟的OP权限等级为2，这意味这它不能执行/ban、/banlist、/ban-ip、/debug、/deop、/kick、/op、/pardon、/pardon-ip、/publish、/reload、/save-all、/save-off、/save-on、/stop和/whitelist命令。
1.9后的版本中，命令方块有了朝向、制约、红石控制/保持开启和三种不同的运行模式。
脉冲模式下的命令方块会在收到红石信号时执行一次命令，直到信号断开并再次激活。
条件制约状态下，只有此命令方块背后的命令方块成功执行了命令，条件制约的这个命令方块才能执行命令。不受制约模式将忽略其他的命令方块。
保持开启将会模拟命令方块一直受到红石激活，并忽略外界的红石信号输入。
上一个输出栏会记录上一次命令执行的结果，无论成功还是失败。右边的O/X按钮可以开启/关闭此栏的功能。当世界中有大量命令方块高速运转时，关闭此栏可以节约内存占用和CPU使用量。
按下GUI界面的完成按钮或是按下键盘上的enter键，可以保存对命令方块的更改并关闭界面。
按下GUI界面的取消按钮或是按下键盘上的ESC键，可以在不保存更改的状态下退出界面。
红石比较器能够输出命令方块上一次执行命令的结果：若上次执行成功，比较器能够输出15级的红石信号。若执行失败则不会输出红石信号。
若命令方块成功执行了命令，它会在聊天栏中向管理员输出结果。在游戏规则commandBlockOutput设置为false时则不会。

概述
哭泣的黑曜石（Crying Obsidian）是黑曜石的变种方块，可通过与猪灵以物易物，废弃传送门的传送门框架及附带的战利品箱，堡垒遗迹的战利品箱获得。
自然生成的废弃传送门中部分黑曜石会被发光的黑曜石替代。
与普通的黑曜石相同，哭泣的黑曜石无法被活塞推动，无法被末影龙摧毁，硬度为 35（仅 Java 版）/50（仅基岩版）。
哭泣的黑曜石可用于合成重生锚，不能用于构建下界传送门。
哭泣的黑曜石会发出亮度等级为 10 的光，会产生紫色滴状粒子效果。

嘎枝之心有概率自然生成在苍白橡树的主干上，使用有精准采集魔咒的工具采集会掉落本体，否则掉落1~3块树脂团（受时运影响）。在快照版本24w44a之前则不会掉落任何物品。
在快照版本24w44a中，亦可使用苍白橡木原木和树脂块合成。
用途
嘎枝之心可以用于生成注册名为 minecraft:creaking_transient 的嘎枝，且这也是召唤该注册名的嘎枝的唯一方式。
当攻击嘎枝时，嘎枝会产生灰色的粒子效果，这些粒子会飘向嘎枝之心，而嘎枝之心也会产生橙色的粒子效果，飘向其生成的嘎枝。
嘎枝之心被激活并生成嘎枝有两个条件：
嘎枝之心的两端必须为朝向正确（与嘎枝之心自身朝向一致）的苍白橡木原木、苍白橡木、去皮苍白橡木原木或去皮苍白橡木；
当前的天气为雷雨或时间为夜晚。
当满足前两个条件时，嘎枝之心会被激活，但不会生成嘎枝。
当满足以上三个条件时，就会激活嘎枝之心，并在水平 16 格、竖直 8 格的范围内生成嘎枝。
激活的 Creaking Heart
破坏嘎枝之心时，其召唤的嘎枝会消失。
如果嘎枝之心已经召唤了嘎枝，破坏参与它激活的原木不会导致嘎枝之心失活。但如果嘎枝之心没有召唤嘎枝，或召唤的嘎枝消失，则该过程会导致嘎枝之心失活。
红石互动
召唤了嘎枝的嘎枝之心可以被比较器检测，输出信号强度取决于所召唤的嘎枝与其的距离。
版本信息
嘎枝之心在 Java 版 1.21.2 快照24w40a 版本和基岩版1.21.50.20 版本中加入。
土径（Dirt Path）是一个装饰性方块，在Java版1.9快照15w31a中加入。它会自然生成于村庄，替代了在以往电脑版用于作为小径的沙石或（1.10及之后）沙砾。
土径的硬度为0.6，它可以被徒手或任何工具开采，但在不同版本中，开采它会有不同结果（见下段）。有锹开采是较快的选择。其中，最快的是金锹，为0.1秒/个。

在电脑版，土径只能通过命令或选取方块放到物品栏里。当被挖掘时，甚至是被带有精准采集的工具挖掘时，它们也只掉落泥土。
在携带版，土径可以被带有精准采集的工具挖掘并掉落自身。
即便如此，土径可以通过对上方为空气的草方块上的侧面或顶部使用任何类型的锹右击来创造。锹会消耗1点耐久。
土径主要用于装饰，它们的颜色和处于沙漠生物群系或热带草原生物群系的草方块相近，然而一个显著的区别就是它们只有15/16个方块高。
土径方块将会在有固体材料置于其上时变成泥土。
备注：携带版中称为“绿茵小道”，1.17前称为“草径（Grass Path）”。
地图（Map）是一种用于观看已探索地形的物品。
显示
合成地图制造出空地图，地图会在它被拿着并按使用键时第一次被绘制。这地图之后能被调节到不同的缩放等级。在转化到绘制的地图物品后，它开始以自上而下的视野绘制玩家周围环境。地图的上方是北方；右方是东；下方是南；左方是西。一个椭圆形的指针指示玩家在地图的位置，并会在玩家穿过地图地形的同时移动。
为了要在地图记录世界，玩家在世界移动时必须拿着该地图物品。世界会在探索时按现状记录，这代表如果世界改变了，玩家必须拿着地图重游该地区来更新地图的视野。地图也能被复制。地图的界限在第一次使用时已固定，这代表地图不会保持不变地以玩家为中心—地图的绘制跟着一个预先设置的网格。
只有在其他玩家的物品栏里放有一张当前地图的拷贝时，他们才能被显示在这份地图上。当把地图放在物品展示框，地图会显示一个绿色的指针，显示物品展示框的位置，这能帮助玩家看见自己在相对地图显示的区域而言的位置。当玩家离开一个地图显示的区域，该地图的玩家指针会变为一个白点。注意如果玩家距离地图的中心太远，该指示物不会显示：半径是每缩放等级320格。该白点会沿着地图的边缘移动来显示玩家的相对位置。
尽管地图在下界能运作，它只会显示红色和灰色的图案。它唯一的用途就是寻找自己在相对制造地图（中心）或放置网格地图（绿色指针）的地方而言的位置。另外，玩家指针在下界会迅速地旋转，所以它并不是好的方向指示器。而末路之地的方向指示器会正常地运作。
玩家能制作朝向上的大型像素画，在那里为中心绘制地图，并把该地图放在物品展示框来制作自定义图画。
地图的每一像素对应世界里能改变大小的区域，它总是对准是8的倍数的X和Z坐标。一般而言，地图像素的颜色和对应区域里从天上看最常见的固体方块的颜色相配。目标区域的“少数方块”对像素的颜色没有影响，因此在比例小的地图，较小的特征是察觉不到的。
地图会以较浅的蓝色显示水下大约15格的地面，所以玩家会看见哪里的地势是升高的。当有陆地在水面上时上述不成立。世界的海拔愈高，地图上的颜色愈浅。即使玩家在地表下移动，地图仍会记录地表上的东西。
尺寸
地图的大小是128×128像素，给予128×128至2048×2048方块（8×8至128×128 区块）的覆盖面，覆盖面取决于缩放等级。
一些相关的距离：128方块（8区块）是主世界玩家的更新半径。然而，在末地和下界是它的一半（64方块）。而且，1024方块是从下界传送门的最少主世界距离，你能在这里建造另一个下界传送门并预期能到达下界的一个新地方。这是穿过一个1:8地图的距离，也是1:16地图从中心到边界的距离。
旗帜标记
JAVA版独有，放置一个旗帜，使用地图右键会出现标记，不会显示花纹，如果重命名过会显示名称。升级地图或破坏旗帜标价会消失。
指针
在Java版中，指示玩家的位置和方向的指针本身就会出现在任何地图里。在基岩版，地图合成时可以包含或不包含位置指针，没有位置指针的地图之后可以通过添加指南针来加入一个，地图还可以使用铁砧给没有指针的地图来添加指针。
缩放
缩小比例在第一次绘制地图（缩放等级0）到最大的大小（缩放等级4）时起作用。地图在全部缩放等级总是对准一个网格，这表示不管地图原本的中心在那里，缩小一个特定范围的任何地图都会有同样的中心点。确切而言，地图会对准地图的宽度（等级3是1024格）减64，在出生点创造的等级3地图会覆盖从-64到960的X和Z坐标。所有在这范围的地图会缩小到同样的坐标，保证它们在地图墙总是“校准”。在基岩版中，地图还可以使用铁砧进行缩小地图比例。
当然，制图台是一个很棒的选择，因为只需一张纸与一份地图即可缩小地图比例。
复制
已探索与绘制的区域会被复制，新探索的区域也会在两个地图都出现。在创造模式，通过对物品展示框里的地图按选择方块能获得复制的地图（确保在选择方块时该地图不在物品栏里，否则该地图只会被移动到可用的快捷栏）。
生存模式下可以使用制图台来复制地图：将一份地图与一份空地图放入制图台即可得到两份相同的地图。
当另一位玩家持有复制过的地图时，地图上会出现两位玩家的定位。
锁定
必须在制图台上使用玻璃板来锁定地图，地图在锁定后会永远保持当前比例与图像，这种方法用来保存自己制作的像素画是一种不错的选择。
