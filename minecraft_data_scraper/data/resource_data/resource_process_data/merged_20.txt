紫颂花（Chorus Flower）是一种可以在末路之地里找到的植物类方块。它是用来种植紫颂植株的必要物品，通常位于自然生成的紫颂树顶端。
紫颂花在Java版1.9快照15w31a中加入。
它们可以被任何工具或徒手开采，并掉落本身。但使用斧的破坏速度更快，其中金斧的速度最快，为0.05秒/个。此外，它还可以被投掷物（弓箭、三叉戟）所破坏并掉落。但如果紫颂花因为被水或活塞推动或是支撑它们的方块消失而被破坏，则不会掉落任何物品。
紫颂花主要用来种植紫颂植株。紫颂花必须种植在末地石或者紫颂植株上，或是水平相邻的一个紫颂植株，且除上方的四个方向必须是空气。否则，紫颂花就会自动破坏且不掉落任何东西。若紫颂花的年龄达到了5（此时它的纹理会呈现紫色而非白色），它将不再生长。玩家可将它收获后重新放置以继续生长。玩家不可通过骨粉催熟紫颂花生长。
注意：紫颂花的年龄从0开始增长，当水平生长时和/或额外长出一朵紫颂花时增加。也就是说，它的年龄并非是按照时间增长的，它的年龄可以在任何时候变为5。位于世界顶部的紫颂花将不再尝试生长。以及，失去紫颂植株支撑的紫颂花会自动破坏而不掉落任何东西，因此只能直接采集紫颂花本身。

红石中继器（Redstone Repeater）在红石电路中是用于“中继”红石信号并放大、阻止信号倒流或者“锁存”信号状态的方块。
红石中继器在Java版Beta1.3中加入。
获得方式
红石中继器可以通过合成、探索丛林神庙或破坏既有中继器获取。每座丛林神庙中自然生成1个红石中继器。
挖掘方式
红石中继器可以用任何或者不用工具立即破坏，总会掉落自身。按挖掘键破坏红石中继器。
红石中继器在下列情况下也会被破坏并掉落自身：
附着方块移动、消失或被破坏；有水流过‌‌（仅限Java版）；活塞试图推动它或者将方块推入其位置。
若熔岩流过红石中继器，中继器会被破坏，不掉落物品。
用途
红石中继器在红石电路中是用于“中继”红石信号并放大、阻止信号倒流、或者“锁存”信号状态的方块。
中继器只能放置在固体方块（例如泥土、石头，反例有玻璃、树叶等）上，也可以放置在倒置的台阶、楼梯与漏斗上。按放置方块键放置中继器。
中继器有前侧与后侧之分——材质上的箭头指向的方向就是前侧。中继器上还有两个小红石火把——火把的颜色指示输出是否为开（输出关时为暗红色，开时为亮红色），两根火把之间的距离表示中继器在信号传输中产生的延迟。
中继器高度为0.125（1/8）格。
充能
中继器传输信号的方向通常只能是从后侧往前侧。
红石中继器可以通过下列元件在后侧输入激活：
1.激活的电源元件（如红石火把、拉杆、红石块）；
2.激活的红石粉；
3.激活的红石比较器或前侧朝向中继器的另一个中继器；
4.激活的固体方块（含任何激活的机械元件，如发射器、红石灯）。
红石中继器可以通过前侧激活下列元件：
1.红石粉；
2.红石比较器或后侧朝向中继器的另一个中继器；
3.任何固体方块（含机械元件）。
红石中继器可以被任何前侧朝向的红石元件以及机械元件激活。
通常称被红石中继器激活的固体方块为“强充能”，而红石粉激活的则称为“弱充能”。强充能的固体方块可以激活临近的红石粉以及其他的红石元件。
信号中继
红石中继器可以“中继”信号，并加强到15强度等级。
红石信号的最大强度为15，信号每通过一格红石粉就衰减1点。若信号不得不穿过15格的红石粉，红石中继器就可以将其加强回满强度。若在中继器前后各放置一块固体方块还可以增加2块传输距离。
信号延迟
虽然红石中继器可以让信号远距离传输，但是由于最小延迟为1红石刻（0.1秒，不含卡顿）所以每次都会产生一些延迟。红石中继器刚被放置时，延迟被设定为1红石刻（等同于2游戏刻，或0.1秒不含卡顿）。
中继器的延迟可以通过对其按下鼠标右键更改。每次使用都会将延迟增加1红石刻，最大为4红石刻，再点击就会降回到1红石刻。更长的延时可以通过多个中继器达成——例如4红石刻与1红石刻延时的中继器可以造成半秒的延迟（0.4秒+0.1秒=0.5秒）。
延迟设置为2至4红石刻的中继器会将正脉冲电路的长度提升到自身的延迟长度。例如，设置为4红石刻延迟的中继器会将1刻、2刻或者3刻的正脉冲电路提升到4刻长，而且不会允许长度4刻以下负脉冲通过。
虽然中继器的延迟不能设置为0刻，但是瞬时中继器电路（无延迟中继信号的电路）可以实现。
作为二极管的特性
红石中继器可以在红石电路中作为“二极管”——中继器仅允许红石信号单向通过（而红石粉与固体方块允许信号双向通过）。
二极管用于保护红石电路免受输出端返回的信号影响或者隔离电路的各区域。
红石中继器的信号锁存
红石中继器可以被另一个连接到其边侧的中继器“锁定”。中继器锁定后，无论输入如何都不会变更其输出（开或关）。当边侧的中继器关闭，中继器的行为会恢复正常。
红石中继器可以被另一朝向它的红石比较器锁定。利用此特性，锁存信号的用途可以更加广泛，特别是因为比较器的输出可以被三侧的输入以及容器影响。
当红石中继器被锁定时，其顶部的小火把显示为一条基岩，指示其状态为锁定。这种现象会在其被另一中继器或处于作差模式的红石比较器锁定时出现。‌‌
在中继器解锁后太快地又被锁定（例如边侧输入来自高频时钟电路）或者锁定信号与输入信号同时更改（例如两个信号由同一组时钟电路通过同延迟的中继器输入）的情况下，中继器不会切换状态。

红石比较器（Redstone Comparator）是能使红石信号流通、比较、作差或检测特定方块状态（主要是容器满度）的红石元件方块。
红石比较器在Java版快照13w01a中加入。
获得方式
按鼠标左键破坏比较器。红石比较器可以用任何工具破坏，并掉落自身物品。
红石比较器在下列情况也会掉落为物品：
1.附着方块移动、移除或被破坏；
2.有水流过；
3.有活塞试图推动它或将方块推入其位置。
若熔岩流过其位置，红石比较器会被破坏但不会掉落物品。
使用
红石比较器可以放置在任何不透明全方块（包括倒台阶与楼梯平面）以及漏斗上。按鼠标右键放下红石比较器。
红石比较器有前后之分——正面的三角指向前方。放下比较器时，比较器会后方（即输入侧）指向玩家。比较器上输入侧有两根小红石火把，靠输出侧则有一根。当比较器的输出大于零时，输出侧的火把会变亮（三角也会变红）。输入侧火把的状态可以由鼠标右键切换：
1.关闭、冻结（指示比较器处于“比较模式”），
2.打开、激活（指示比较器处于“作差模式”）。
红石比较器可以从后侧及左右侧接受信号强度。边侧输入只能从红石块、红石粉、红石中继器和其他比较器接收。红石比较器的前侧是输出端。
信号流穿红石比较器需要1红石刻（2游戏刻或0.1秒，不考虑延迟），不论输入端是前侧还是边侧。此规律对于信号强度或开与关的充能都适用。红石比较器通常不会持续1刻的信号波动或强度变化——例如单刻脉冲的发生器在侧边输入时会被认为总是关，在后侧输入则是总为开。
红石比较器有四大用途：承载信号强度、比较信号强度、信号强度作差以及检测方块状态（主要为容器的满度）。
承载信号强度：
红石比较器侧边没有输入时，仅会输出与后侧相同的信号强度。
比较信号强度：
红石比较器处于比较模式时（前侧火把关闭）会将后侧输入与边侧输入比较。若有任意边侧输入大于后侧输入，比较器输出关闭。若两边侧都不大于后侧输入，则比较器仅会输出与后侧相同的信号强度。
信号强度作差：
红石比较器处于作差模式（前侧火把打开）会输出后侧输入信号与两边侧中较强的信号差（最小为0）。
例如，若后侧输入信号强度7，左侧输入2，右侧为4，那么输出则为3信号强度，即7 - 取较大(2,4) = 3。
检测方块状态：
红石比较器会将后方的方块看做信号源并根据方块的方块状态输出对应信号。比较器与受测方块可以被固体方块隔离。
用于检测容器状态的红石比较器会根据物品装满的程度输出对应信号（空为0，满为15，以此类推）。
下列容器可以被比较器检测：
熔炉、高炉、烟熏炉、酿造台、漏斗、探测铁轨上的漏斗矿车、发射器、投掷器、箱子、陷阱箱、探测铁轨上的运输矿车、木桶、大箱子、大陷阱箱、潜影盒。
当比较器检测大箱子或大陷阱箱时，比较器会检测整个大箱子（54格），而不是比较器后方的那半个。不能打开的箱子（无论是因为上方有固体方块还是有豹猫在上方）总会使比较器输出0，其中的物品数量不影响。
右边的容器信号强度所需最少物品表格展示了各容器中，64为1组的物品需要多少组才能输出指定的信号强度值。“组”前的数字表示满叠的物品需要组数（不需要1组则不显示），“个”前的数字表示整叠物品以外需要的物品数量（没有零头时不显示）。对于只能堆叠到16个的物品（雪球、告示牌、末影珍珠等），“个”前的数字除以4取整就可以适用。对于不可堆叠的物品，每1“个”都表示额外物品。
例如，需要依靠漏斗产生10强度的信号需要至少3满堆叠又14物品（或16每组的物品，则为4组物品）。
已知物品数量，求信号强度值
容器空时，输出关；容器不为空时，输出按下列计算：
信号强度 = 向下取整[1 + (所有物品栏的满度和 / 物品栏总数) * 14]
单格的满度 = 单格中的物品数 / 此类物品最大堆叠数
注意不可堆叠的物品总被认为充满一格（1物品，1最大堆叠数：1 / 1 = 1.0），最多堆叠到16的物品（例如末影珍珠、雪球或鸡蛋）也类似地看作16即为充满。
已知信号强度值，求物品数量
这对于利用比较器与容器配合来输出特定强度来说是有用的。输出特定信号强度所需物品数量用如下算法计算：
所需物品数 = 取较大{目标信号强度, 取整[(容器总格数 * 64 / 14) * (目标信号强度 - 1)]}
一些非容器的方块也可以被比较器检测：
蛋糕：
蛋糕输出的信号强度取决于蛋糕剩余量。每片输出2强度，总共有7片，也就是说完整蛋糕输出14强度。
炼药锅：
炼药锅输出的信号强度取决于水量。从完全无水到完全充满，输出分别为0、1、2与3。
命令方块：
命令方块存储最后执行的命令的“成功计数”，表示此命令方块最近执行成功命令的次数。“成功”的定义由命令的成功条件决定：如果聊天界面显示红色错误信息，那么命令执行不成功。
大多数命令每次执行只能成功一次，但特定命令（例如将玩家作为参数的命令）可以成功多次，比较器会输出全部的成功数（最大15）。
命令方块持续存储最后执行的命令的成功数，直到再次执行命令。因此比较器会持续输出同一信号强度，即使命令方块已经不在激活状态（命令方块关闭不影响比较器开关）。
末地传送门框架：
末地传送门框架有末影之眼的时候输出满强度15，若非则0。
物品展示框：
比较器可以检测物品展示框的内含物状态。若要检测展示框的状态，比较器必须连接在展示框附着方块的后方，并后侧朝着方块。若展示框内无物品，则输出0，此外则根据物品旋转角度为1至8强度（1为默认方向，每次右击都旋转直到8，然后再返回1）。
唱片机：
唱片机输出的信号取决于正在播放的唱片。对应关系请见上方的容器信号强度所需最少物品表格。

红石火把是可反相（熄灭）的红石信号源、光源，是一种非固体方块。
红石火把在Java版Alpha v1.0.1中加入。
生成、放置与破坏
红石火把在雪屋中自然生成。
红石火把可以附着于：
1.不透明全方块（石头、泥土等）的顶部或侧面，包括全方块的红石元件（命令方块、发射器、投掷器、音符盒、红石灯）；
2.特定方块的顶部：漏斗、苔石墙、栅栏、玻璃、地狱砖栅栏、倒置台阶或楼梯。
红石火把不能放置在任何方块的底部。尝试在无效的表面附着红石火把会使其“滑入”这块空间内最近的有效表面。例如地面上的栅栏，尝试在栅栏的侧面放置红石火把会使其附着到栅栏周围地面方块的顶部。
不使用工具即可破坏红石火把，并令其以物品形式掉落。
红石火把在下列情况也会掉落为物品：
1.红石火把所附着的方块被移动、移除或破坏；
2.有水或熔岩流经红石火把所在位置（当熔岩流经时，红石火把在掉落后会被立即烧毁）；
3.有活塞试图推动红石火把，或将其他方块推入红石火把所在位置。
用途
作为合成材料，参与合成激活铁轨、红石比较器与红石中继器。
作为红石元件产生强度为15的红石信号，常用于激活各类红石元件（如红石粉、活塞），也参与构成大部分红石逻辑门（与门、非门、或非门、与非门、异或门、同或门、蕴含门等）。
作为光源，亮度等级为7，是除棕蘑菇外可发光物体中最暗的，不能融化雪或冰，也不能防止怪物生成（尽管它能降低怪物的生成概率，且能防止怪在它本身这一格生成）。在非创造模式下，往往只会出于装饰和少数其他原因才会将红石火把用作光源。
激活与熄灭
红石火把有激活（点燃）和未激活（熄灭）两种状态，激活的红石火把会：
1.充能临近的红石粉至15能量等级，“临近”包括附着在方块侧面的红石火把的下方；
2.充能临近背朝红石火把的红石比较器、红石比较器至15能量等级；
3.强充能其上方的不透明固体全方块（但不能是侧面或下方的）；
4.激活临近的机械元件，例如红石灯等；
5.产生强度为7的亮光，产生颗粒视觉效果。
当红石火把在所附着的方块未被充能时，红石火把就将保持激活（点燃）。换言之，红石火把将所附着方块被充入的信号反相：0能量等级会反相为非0（15），非0能量等级会被反相为0（若想得到其他能量等级的输出，应配合作差模式的红石比较器等其他红石元件）。
由于墙、栅栏、玻璃、台阶和楼梯都无法被充能，因此附着于这些方块的红石火把不能被关闭。特别的，漏斗虽然能接受红石信号充能，但附着于漏斗的红石火把也不能被关闭。
红石火把永远不会影响其附着的方块，即使它是红石元件。例如，红石火把附着的红石灯不会被激活。
红石火把的点燃和熄灭过程耗时1红石刻（2游戏刻，或在不考虑卡顿时0.1秒），其附着方块的信号变更至少需要持续1.5红石刻才能让红石火把改变状态（除非前者的变更由在短路电路中的红石火把自身引起），因此在一些情况下，比如当红石火把在自己的附着方块受到1刻及更急促的脉冲时不会关闭。
当红石火把在点亮与熄灭状态之间切换得过于频繁（100个单位时间(大约5秒)之内达到8个以上点燃-熄灭周期）时，火把会“烧毁”，即保持熄灭。这通常发生在玩家有意或无意地将一个红石火把与它自身相连，也即建造了一个无延迟反馈回路的时候，这会让红石火把迅速进行点燃和熄灭的切换，任何该与火把相关的元件有可能都被快速地激活与关闭，当受影响的元件过多时，很可能导致游戏的卡顿甚至崩溃，这也许正是Notch编程让火把会烧毁的原因。红石火把烧毁时会冒出一缕烟并发出像火把被浇灭时那样嘶嘶声，并且保持熄灭；直到100个单位之间内只有7个及以下的点燃-熄灭周期，并且在相邻的方块上产生一个释能事件（比如新下降沿或一般意义上的方块刷新），才能重新激活烧毁的火把。单个红石火把没有烧毁次数的上限。
（以上资料部分摘自中文Minecraft Wiki）
红石逻辑门
红石火把是大部分红石逻辑门的重要组成部分，关于红石逻辑门的官方资料见中文Minecraft Wiki中的逻辑电路词条。
需要用红石火把建造的红石逻辑门如下：
①非门
需要最少材料：1个固体方块、1个拉杆、1个红石火把（红石灯仅供显示输出效果，没有实际作用，同下）
非门拥有一个输入端A和一个输出端B。
当A的信号为0（拉杆没有拉下）时，B输出的是1（红石灯开启）；当A的信号为1（拉杆被激活）时，B输出0（红石灯关闭）。
②与门
需要最少材料：3个固体方块、2个拉杆、3个红石火把、1个红石粉
与门拥有2个输入端A1与A2，以及1个输出端B。
必须A1与A2都为1时，B才能为1；否则B都为0。
③或非门
需要最少材料：3个固体方块、2个拉杆、1个红石火把、3个红石粉
或非门拥有2个输入端A1和A2，1个输入端B。
必须A1与A2都为0时，B才能为1；否则B都为0。
④与非门
需要最少材料：4个固体方块、2个拉杆、2个红石火把、2个红石粉
与非门拥有2个输入端A1和A2，1个输出端B。
必须A1与A2都为1时，B才能为0；否则B都为1。
⑤同或门
需要最少材料：10个固体方块、2个拉杆、8个红石火把、8个红石粉
同或门拥有2个输入端A1和A2，1个输出端B。
当A1与A2都是0或都是1时，B才能为1；当A1与A2其中一个是0，一个是1时，B为0。
⑥异或门
需要最少材料：9个固体方块、2个拉杆、7个红石火把、7个红石粉
异或门拥有2个输入端A1与A2，1个输出端B。
当A1与A2都是0或都是1时，B为0；当A1与A2其中一个是0，另一个是1时，B为1。
⑦蕴含门
需要最少材料：4个固体方块、2个拉杆、1个红石火把、3个红石粉
蕴含门拥有2个输入端A1与A2（插有红石火把的一端是A1），1个输出端B。
当A1为1，A2为0时，B为0；否则B都为1。
上述逻辑门中的拉杆可以用其它充能装置代替，而红石灯也可以用红石线路等代替。
红石灯（Redstone Lamp）是一种在被红石机关激活后能发光的方块。
红石灯在Java版1.2.1快照12w07a中加入。
红石灯可以徒手或用任何工具开采。其本身将掉落成为一个物品。
红石灯是一种可控光源，属于红石元件，可被下列红石元件激活：
一个相邻的电源，包括顶面和底面，例如红石火把（但不会激活它直接附着的红石灯）、红石块、阳光探测器。
一个相邻的充能的方块，例如一个底部有红石火把的非透明方块，包括顶面和底面。
一个充能并指向红石灯的红石比较器或红石中继器。
相邻的充能红石粉，方向指向红石灯（或在其顶面）或没有指向；红石灯不会被不指向它的临近红石粉激活。
红石灯会立刻被激活，但需要2红石刻来关闭（4个游戏刻，也就是滞后0.2秒）。
激活的红石灯会产生15级的光照。未激活的红石灯不提供光照。
虽然红石灯在大多数情况下拥有非透明方块的特征（火把与拉杆可以被依附于它，红石粉可放置其上，等等），但激活的红石灯对于天空亮度是“透明”的，只减少1等级的天空亮度。

红石粉
红石粉（Redstone Dust）是一种放置后可以传递红石信号的物品。使用铁以上硬度的镐（没有精准采集附魔的）挖掘红石矿石或深层红石矿石掉落，时运附魔可以增加掉率。用于酿造药水（延长效果）、合成红石相关物品以及放置为红石线以组成红石电路。
红石粉在Java版Alpha v1.0.1中加入。1.14快照18w43a前红石粉的纹理
可以放置于全高不透明方块或上半砖的顶部（完整方块面），放置后会自动连接周围相邻的红石元件或者高度差不超过1米的其他红石线（除非被方块边缘阻挡），一旦发生连接就会从点状变为线状并指向连接处。
红石线可以接受并传递红石信号，红石信号每经过一个红石线到达下一个红石线时强度衰减1，不同强度的红石信号同时经过一个红石线时，只传递信号强度更强的信号。红石线接受到红石信号时呈亮红色，颜色会随着信号强度减弱逐渐暗淡。
红石线向其他红石线传递红石信号是一种“强充能”，这时接受信号的红石线可以继续将信号传递给其他红石线或红石元件。而当红石线接受到红石信号时，红石线所在的方块以及红石线呈线状时线端点处连接的不透明方块会被“弱充能”至相同强度红石信号，这是指红石信号无法通过这个方块传递给与之相连的其他红石线，但仍可以传递给与之相连的某些红石元件，下图中的两个圆石就是被弱充能的。可以利用这个性质制作穿墙电路。
在1.13版本之前，红石粉（Redstone Dust）的名称是红石（Redstone）。
在1.13版本扁平化之前，红石（方块）的数字id是55，红石（物品）的数字id是373。

红色下界砖块（Red Nether Bricks）是一种下界砖块的红色变种。
获得方式
红色下界砖块可以通过下界疣和下界砖合成得到。
它需要被任何镐开采。若不用镐开采，则不会掉落任何东西。其中，金镐开采的速度最快，为0.25秒/个。
用途
红色下界砖块与下界砖块都是不可燃方块，且对恶魂火球免疫，这使得它成为合适的下界庇护所材料。

介绍
细雪是一种会对生物和玩家造成伤害的方块。
生成
细雪会自然生成于雪林生物群系的地表，同时也会自然生当成在试炼密室的部分结构中，铺满地面。
用途
当玩家踩在细雪上时，玩家会很快陷入细雪中，同时跳跃能力降低，且玩家的生命值会变成青蓝色冻结纹理，HUD也会出现相同纹理，并在7秒（140ticks）之后进入冻伤状态，同时玩家的身体会颤抖，并且每2s（40ticks）就会减少1点（）生命值，生物同理（对于烈焰人、岩浆怪、炽足兽则是5点（）），装备了皮革铠甲的生物或玩家可以防止此伤害。
生命值的冰冻动画
极限模式中生命值的冰冻动画
玩家或能够穿上装备的生物（如僵尸）穿上皮革靴子时不会陷入细雪中。马穿上皮革马铠后同样如此。
骷髅处于细雪中7秒（140ticks）后会开始转化为流浪者，再经15秒（300ticks）转化完成。转化成流浪者后，其生命值会回满，且其原本携带的装备仍会保留。
山羊不会生成在细雪里，而且山羊也会避开细雪。
细雪可以消除生物的着火状态，在此同时细雪会被破坏。
获得方式
细雪在创造模式物品栏内没有物品形式，对着细雪按下选取方块键获得的是细雪桶。
细雪被挖掘时不会掉落物品，即便是使用带有精准采集魔咒的工具。任何工具都不能加速破坏细雪。
细雪会被流过的水或熔岩破坏。
版本信息
细雪在 Java 版 1.17 快照20w46a 版本和 基岩版  1.16.200 预览版1.16.200.52 版本中加入，此时在基岩版属于实验性玩法。
在 基岩版1.16.200 正式版中，细雪不再属于实验性玩法。
细雪桶是一个充满着细雪的桶。
细雪桶在Java版1.17快照20w46a中加入。
获得方式：
手持桶对细雪点按鼠标右键来得到细雪桶，细雪随之被消耗。
也可以通过在下雪天使用炼药锅接雪来为炼药锅装满细雪，在炼药锅满后即可用桶取出。
使用：
手持细雪桶对固体方块右击，细雪将被置于其上，同时细雪桶还原为空桶。
发射器可被用来装载细雪桶。当其被激活时，细雪会在其正面被放置，同时其内的细雪桶被清空；此时再次激活，所放置的细雪将被回收进发射器内的空桶。
炼药锅可被一个细雪桶填满，装满细雪的炼药锅也可以装满一个细雪桶。
当玩家在细雪上行走且不陷进去会获得进度【轻功雪上飘】。

注：本条目介绍的是绊线，而不是绊线钩或线，请注意写法，是“绊线”而非“拌线”！
简介
绊线（Tripwire）是一种可以触发绊线钩红石机关的方块。
绊线可以由线放置，与绊线钩组成绊线线路。
生成
丛林神庙会自然生成 5 个绊线，试炼密室的 arrow_dispenser 结构会生成 10 个绊线，encounter_4 结构会生成 3 个绊线。
用途
对着要放置绊线的方块上使用线，即可放置出拌线。与红石电路一样，可以拐弯放置。
绊线可用于构成绊线电路：在一条直线上，中间为 1～40 个绊线，两端为附着在方块上的绊线钩。若为有效结构，绊线的碰撞箱会变矮：顶部 0.09375 或 3/32 格，底部悬空 0.0625 或 1/16 格。而一般的绊线碰撞箱为 0.5 格高且不悬空。
几乎所有实体（除某些箭、喷溅型药水、末影珍珠和末影之眼）都会激活处于绊线电路中的绊线，激活后每 10游戏刻更新一次，若没有实体接触就会取消激活状态（因此实体离开时不会立刻更新）。绊线被激活或被破坏（不包括使用剪刀破坏）（在基岩版中不包括水流破坏）会触发绊线钩 5红石刻（10游戏刻，0.5s），绊线并不会产生红石信号。不在绊线电路中的绊线也会被激活，但是只能通过 BUD 和侦测器检测到。
版本信息
绊线在 Java 版 1.3.1 快照12w22a 版本和携带版 Alpha0.13.0 build 1 版本中加入。
