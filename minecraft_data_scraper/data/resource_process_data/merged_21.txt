结构方块是用来手动创建结构的方块，在 Java 版 1.9 快照 15w31a 中加入。
结构方块和结构空位只能通过 /setblock 或者 /give 命令来获得。结构方块和基岩、命令方块一样，不可以被处于生存模式下的玩家破坏，也不会被爆炸炸毁。
仅能在创造模式打开结构方块的GUI。
结构方块包含4种工作模式：
上图从左到右依次为：
Data（数据）储存箱子标记和其他需要数据的方块。
Save（保存）保存记录方块和实体的位置，并且储存它到文件中（默认）。
Load（加载）加载保存的结构文件。
Corner（角落）自动计算结构的尺寸。
注：在基岩版中仅包含保存和加载两种模式因为正常情况下这两种就够用了，且材质不与Java版完全对应。
数据模式
数据模式用于为战利品箱分配战利品表。当使用结构方块强制加载原版/模组文件中带有战利品箱的结构时，存有对应战利品表数据的数据结构方块会伴随战利品箱出现。
使用结构方块加载的沉船中的数据结构方块
使用结构方块加载的暮色森林模组中德鲁伊小屋的地下室，其中箱子的位置被数据结构方块占据
保存模式
保存模式用于保存框定的结构，被保存的结构文件储存于".minecraft\saves\【存档名称】\generated\minecraft\structures"中，可通过传输对应文件实现跨存档转移结构。
保存结构方块的GUI：
结构名称：对应结构的命名。
相对位置：框选区域东南靠下的角落相对结构方块的坐标，从左到右依次表示x、y、z轴。
结构大小：框选区域的大小，从左到右依次表示x、y、z轴上的范围。
（注：相对位置和结构大小的数值不能超过32）
显示隐形方块：开启后会显示框选区域内的空气和结构空位。蓝色立方体是空气，红色立方体是结构空位。
探测结构大小和位置：需配合角落结构方块使用。
包括实体：开启后会保存区域内的实体。
结构方块会使用白色边框（其中三条边为红、绿、蓝三种颜色，分别表示x、y、z轴向，三轴交汇的角为东南靠下角，即结构区域的中心）表示框选区域。框选结构
点击GUI中的“保存”按钮或给予结构方块红石信号以保存结构。
加载模式
加载模式用于从文件中加载结构。
加载结构方块的GUI：
结构名称：欲加载结构的名称。从原版或模组文件中加载结构相对繁琐，此处不单独介绍。
相对位置：欲加载结构的结构中心（未选择选择或镜面时为东南偏下角）与结构方块的相对位置。
显示边框：关闭后不再显示欲加载结构的大小边框。
结构完整性及种子：结构完整性为加载后结构相对原文件的完整性，1为全保留，0.5为每个方块有1/2几率保留，等等。种子决定了方块是否保留的算法初始数，同一结构、相同完整性在种子相同的情况下加载结果相同。
包括实体：关闭后不再加载保存在对应结构中的实体。
旋转与镜像：模式按钮后方的五个按钮。四个数字表示旋转角度，旋转方向为顺时针。中间的符号表示镜像，“|”表示不镜像，“<>"表示南北镜像，”^v"表示东西镜像，镜像操作先于旋转操作。
（注1：在选择了旋转或镜像后，三彩色轴的交汇处可能不再是东南靠下角，但一定是结构的中心）
（注2：旋转与镜像都是以结构中心为原点进行的）
点击GUI中的“加载”按钮，若结构名称无误将显示准备就绪的信息，并出现结构区域的线框，若关闭显示边框则不会出现边框。
再次右击或通入红石信号即可加载结构。未勾选包括实体的情况下加载的结构
角落模式
角落模式用于框选结构区域。
只需将两个命名相同的角落结构方块放在欲框选区域的两个对角，并在一定范围内放置命名与其相同保存结构方块，即可使用“探测结构大小和位置”功能快速定位该结构。其实一般情况下放置这两个角落结构方块还没自行输入数值方便使用角落结构方块框选区域
使用角落结构方块保存时亦不能超出32格的范围。
如果一定要保存超过32格的结构，可以使用多个结构方块分部分保存，且将子结构方块框选在父结构方块的区域内，并配上一个侦测器，即可实现连续加载。
结构空位用于结构方块的保存模式中，在粘贴的时候不会替换原本的方块。举个例子：你在陆地上复制了一艘船，如若直接复制粘贴到海中因为复制时船旁边有空气将原本的水方块替代从而使水缺失且流动，十分不美观，但是如果将结构空位放置在船旁边就会防止这一现象。
结构空位在Java版1.10快照16w20a中加入。

花岗岩（Granite）是一种生成在地下的方块。
生成
花岗岩会以团簇的形式生成在主世界Y＝0～60及64～128的范围，前者每区块生成2次，后者尝试生成6次且会有5/6的概率生成失败，大小为0～64个方块。
花岗岩也会和铜矿石生成在Y＝0～50的范围，形成铜矿脉。
获得方式
花岗岩也可通过使用闪长岩和下界石英合成获得。
用途
除装饰外没有任何用途。
版本信息
花岗岩在 Java 版 1.8 快照14w02a 版本和 携带版 Alpha 0.9.0 预览版build 1 版本中加入。
在 Java 版 1.13 快照17w47a 版本扁平化前，数字 ID 为1。
在 Java 版 1.14 快照18w43a 版本和 基岩版 1.10.0 预览版1.10.0.3 版本中修改了纹理。
在 Java 版 1.18 快照exp4 版本中，石岸地表有时候会生成条带状的花岗岩地层，而在 Java 版 1.18 快照21w37a 版本中回退了这一改动。
花盆（Flower Pot）是一种内部可以放置植物的装饰性方块，在Java版1.4.2快照12w34a中加入。
花盆可用任何工具瞬间开采。
花盆在被活塞推动或有水流过时会掉落自身（花盆中的植物也会掉落）。
花盆可用来保存不同的植物。蘑菇、任何1格高的花、枯死的灌木、仙人掌、树苗、蕨、竹子、菌类（包括绯红菌和诡异菌）、菌索（包括绯红菌索和诡异菌索）、杜鹃花丛（包括杜鹃花丛和盛开的杜鹃花丛）和红树胎生苗都可被放置在花盆的里面。
（除携带版外）花盆也可放在上下倒置的台阶和楼梯。花盆高3/8格（即0.375格），且可以站上去。不用跳跃的方法从花盆的顶部走向一个全体积方块是不可能的，但从花盆跳上栅栏是可以的。
（1.11及之后）植物可以通过对花朵使用使用键移除。
花盆可用于展示仙人掌或凋零玫瑰而不继承它们危险的特性。
花盆可以通过如下渠道获得：
（1.4.2及之后）通过3个红砖合成。
（1.4.2及之后）沼泽小屋会生成一个装着红色蘑菇的花盆。
（1.9及之后）雪屋的地下室有一个装着一个仙人掌的花盆。
花盆可用任何工具或不同工具进行开采，在被活塞推动时会掉落自身。
