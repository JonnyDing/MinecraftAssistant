简介
收纳袋（Bundle）是一种用于存储其他物品以便管理的物品，通过提示框来作为交互界面。
获取
在 Java 版 中，1.17 版本只能通过指令获取。在加载洞穴与山崖预览数据包后，收纳袋则能够合成。1.18-exp1 版本后，收纳袋可以正常获得和合成了。在 1.21.2 快照24w33a 版本中再次修改了收纳袋的合成配方，使其在前期生存中更容易被获取。
在基岩版中，1.21.30 预览版1.21.30.23 版本以前，只限于开发者版本中使用，在1.21.30.23 版本中，收纳袋为实验性玩法“同捆包”（基岩版的错译），并且可以合成。
用途
收纳袋可以在物品栏中收集储存杂物，最多可以储存等同于 1 组的物品，相当于内部有 64 个槽位，每个单组堆叠上限为 64 的物品占用 1 个槽位，每个上限为 16 的物品占用 4 个槽位，不可堆叠物品、装有蜜蜂的蜂巢或蜂箱（1.21.2 快照24w33a 版本后，仅 Java 版）占用 64 个槽位。当然，收纳袋不能储存潜影盒。
虽然收纳袋本身不可堆叠，但每个收纳袋本身只会占用 4 个槽位，再加上内部已储存的物品。即每个收纳袋最多可以放入 16 个空的收纳袋。
从收纳袋中放入物品，可以将物品拖到收纳袋上方后，使用右键装进收纳袋；或者将收纳袋放在物品上方使用右键。要取出物品，需要右击收纳袋或按住右键在物品栏中拖动。在快捷栏中选中收纳袋时，按下使用键会扔出其中所有的物品，而在物品栏中拖动时，后放入的物品会先释放出来。
在 Java 版 1.21.2 快照24w34a 版本和基岩版 1.21.30 预览版1.21.30.24 版本后，收纳袋的提示框会显示其内容物，最多能查看8个槽位。被高亮的物品（若有）表示右键点击收纳袋时会被取出的物品。如果还未装满，则会在末尾显示数个空格子。如果收纳袋中收纳了超过 8 种不同的物品，那么在第七个物品后面会显示”+…“来表示目前收纳袋里有多少种不同的物品，并且除非拿出前面的物品，否则不能再通过普通方式拿出后方”+…“的物品。可以滑动鼠标滚轮切换下一个取出的物品。
当收纳袋以掉落物的形式被炸毁或烧毁时，其中储存的所有物品都会被释放。
版本信息
收纳袋在 Java 版 1.17 快照20w45a 版本和基岩版 1.18.10 预览版1.18.10.22 版本中加入 ，在基岩版中仅限于开发者版本使用。
在 Java 版 1.17 快照21w19a 版本中，在创造模式物品栏中和收纳袋的合成方式移除，在1.18-exp1 版本中重新加入，在 1.18 快照21w37a 版本中再次在创造模式物品栏中和收纳袋的合成方式移除，在 1.19.3 快照22w42a 版本中重新加入，属于实验性内容，在 Java 版 1.21.2 快照24w33a 版本中更改了合成方式，在24w39a 版本中不再属于实验性内容。
在基岩版 1.21.30 预览版1.21.30.23 版本中，收纳袋属于实验性玩法，在 1.21.40 预览版1.21.40.23 版本中不再属于实验性玩法。
在 Java 版 1.17 快照20w46a 版本、1.21.2 快照24w33a 版本和基岩版 1.21.30 预览版1.21.30.23 版本、1.21.40 预览版1.21.40.23 版本中修改了纹理。
在 Java 版 1.21.2 快照24w38a 版本和基岩版 1.21.40 预览版1.21.40.22 版本中，在实验性内容和实验性玩法中增加了 16 种颜色的变种，都在下一个版本中不再属于实验性内容和实验性玩法。

故障叶是一种渲染存储树叶模型的方块，是基岩版和携带版特有的方块，方块和物品 ID 为 254，在携带版 Alpha 0.2.1 版本加入，在 0.8.0 build 1 版本移除。
故障叶只能通过编辑物品栏获得。
故障叶在被剪刀挖掘时掉落普通树叶。
纹理
元数据为 0 的故障叶对应橡树树叶，但是比橡树树叶要深一点，且不会枯萎，用#48b518着色；
元数据为 1 的故障叶对应云杉树叶，但是侧面是工作台的纹理，用#619961着色；
元数据为 2 的故障叶对应白桦树叶，用#80a755着色；
元数据为 3 的故障叶对应丛林树叶，但由于 leaves_carried 的移除，着色颜色未知。

故障草是一种渲染存储草方块模型的方块，是基岩版和携带版特有的方块，方块和物品 ID 为 253，在携带版 Alpha 0.5.0 版本加入，在 0.8.0 build 1 版本移除。

纹理

旗帜在 Java 版 1.8 快照 14w30a 中加入。
旗帜是一种装饰性方块，可以通过工作台或织布机向旗帜添加图案（消耗染料，部分图案需要特殊物品。例如花朵图案，在工作台添加时需额外添加滨菊，而在织布机添加时需要花朵旗帜图案），同一面旗帜可添加多个图案，新图案直接覆盖在旧图案上方，最多叠加 6 层图案， 图案数量＞6 个的旗帜仅能通过指令获取。
用织布机向旗帜添加图案时，左侧会出现各种图案的小图标，选中所需图案样式，并向右上方的格子中依次放入旗帜、染料、旗帜图案（非必要），右侧图案效果，从右下方输出成品；用工作台向旗帜添加图案时，需将多个染料摆成所需图案进行有序合成，不同的摆放方式对应不同的图案样式。
用有图案的旗帜右键装水的炼药锅可以洗去最后一次添加的图案，每次洗涤都会使炼药锅中的水会减少一层。
旗帜可以与盾牌合成，以此在盾牌上附加图案。（附加的图案会被缩小，因此在一定程度上失真）。
用地图右键旗帜，可以在地图上添加一个标记，标记颜色与旗帜底色相同，若旗帜在铁砧中重命名过，标记将显示相应的名称。若旗帜方块被破坏，未经制图台锁定的地图将同步移除该旗帜的标记。
有图案的旗帜与另一底色相同的空白旗帜无序合成时（此处“底色”指最初合成旗帜时使用的羊毛的颜色），可以把图案直接复制到空白旗帜上，无需消耗染料。
作为燃料在熔炉中使用时，每个旗帜可以燃烧 1.5 个物品。

在 Java 版中，灾厄旗帜是一个有总共 8 个图案的白色旗帜。
低配电脑请谨慎使用，因为旗帜属于实体方块，占用性能。
猪鼻图案。只能在堡垒遗迹的箱子里找到。
旗帜图案（Banner Pattern）是用于在织布机上自定义旗帜的物品，替代了原来在工作台上合成自定义旗帜的方式，使自定义旗帜变得更加直观和简便。在Java版中，它们有6个变种；在基岩版中，它们有8个变种。

无限书盒（Box of Infinite Books）是一种可以生成随机成书的方块，自然生成于“library”和“isolation”维度中。
无限书盒的 5 个面都使用橡木木板的纹理，而在另外一面是根据坐标随机分配的纹理，共有 4 种颜色的纹理，在纹理文件夹中的顺序依次为黄、蓝、红、绿。其中蓝色书盒是显示在物品栏中的纹理。
破坏时会掉落本体。
右击无限书盒时，会生成一本成书，成书的内容是随机的，但同一个无限书盒只会生成一本固定内容的成书。
成书共有 16 页，作者均为 Universe itself，标题格式为：<区块X>/<区块Z>/<方块方向>/<本区块边缘至该方块右侧的距离>/<方块Y>。
若游戏规则 doTileDrops 为 false，则不会生成成书。
时钟（Clock）是一种用于观看时间的物品。
时钟在Java版Alpha v1.2.0版本加入。
观察它的颜色变化和旋转角度，即可知道当前游戏时间为白天还是黑夜。
在主世界，时钟在物品展示架上、掉落物形式、物品栏/快捷栏、手中以及其他地方都会显示时间。
注意：时钟在下界、末路之地均无法正常工作，有许多模组添加的维度中时钟也会失效。
关于译名：1.13及以前的译名为“钟”，1.14预览版18w44a加入了钟（Bell），为了防止冲突，故更改现译名为“时钟”。
专家级别的图书管理员村民有50%的几率以5个绿宝石的价格出售1个时钟。‌‌

在1.16快照20w19a之前，配方中的时钟也可以显示时间，因此无需合成时钟便可以看时间。但在1.16快照20w19a及以后，此方法已失效。
智能手表（Smarter Watch）是一个可以统计玩家一些信息的智能物品，仅能在创造模式物品栏中获取，无法合成。
智能手表仅佩戴在玩家的副手上才会生效，此时会在快捷栏的物品提示文本中显示现在的时间，采用了 24 分钟一天的机制，但由于一个游戏日为 20 分钟，所以会出现错误时间。
每次接受信息时会触发音效。
智能手表会记录玩家行走距离、飞行距离、距上次死亡时间、使用物品次数、击杀生物等统计信息。
暗海晶石是一种海晶石的变种。它是一种带有网格图案的暗色方块。在海底遗迹中，暗海晶石较少, 它通常包围着 8 个金块，并且有很小几率作为装饰出现。
它的爆炸抗性为 30，硬度为 1.5。需要用镐来开采它才能掉落它自身。其中，金镐是最快的，为 0.2 秒/个。
